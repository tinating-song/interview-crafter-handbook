# ATS友好型简历指南

**重要性：** ⭐⭐⭐⭐⭐ | **目的：** 让你的简历通过机器筛选

## 📌 什么是ATS？

ATS (Applicant Tracking System，申请人跟踪系统) 是企业用来筛选简历的软件。

```
        简历提交
            ↓
    ┌───────────────┐
    │  ATS 系统     │ ← 你的简历在这里被扫描
    └───────────────┘
            ↓
    ┌─────────────────────────┐
    │  简历评分和排序          │
    │  ✅ 匹配度高的 → 人工审核 │
    │  ❌ 匹配度低的 → 自动拒绝 │
    └─────────────────────────┘
```

**关键数据**：
- 75%的简历被ATS自动淘汰，无人看到
- 大公司99%使用ATS筛选
- ATS平均扫描一份简历：6秒

## ✅ ATS友好格式

### 1. 文件格式

| 格式 | ATS兼容性 | 推荐度 |
|------|----------|--------|
| .docx | ⭐⭐⭐⭐ | ✅ 推荐 |
| .pdf (文本型) | ⭐⭐⭐⭐ | ✅ 推荐 |
| .pdf (图片型) | ⭐ | ❌ 避免 |
| .pages | ⭐ | ❌ 避免 |

**最佳实践**：提交PDF时确保是文本可选择型，不是扫描图片

---

### 2. 简历结构

#### 标准顺序
```
1. 联系方式 (顶部)
2. 专业总结 (可选)
3. 技能 (紧靠联系方式)
4. 工作经历 (倒序)
5. 教育背景
6. 项目经历 (可选)
```

#### ❌ 避免的内容
- 照片（ATS无法识别，还有歧视风险）
- 图表/图形（文本提取失败）
- 表格（可能导致格式混乱）
- 多栏布局（阅读顺序可能错误）
- 页眉页脚（内容可能被忽略）

### 3. 关键词优化

ATS通过关键词匹配来筛选简历。你需要**从JD中提取关键词**并放入简历。

#### 如何提取关键词？

**步骤1：分析JD中的关键词类型**

```
职位描述示例：
"我们正在寻找资深后端工程师，
负责高并发系统设计。要求：
- 熟练使用Java、Spring Boot
- 有微服务架构经验
- 熟悉MySQL、Redis、消息队列
- 了解Docker、K8s"
```

**提取的关键词**：
- 技术栈：Java, Spring Boot, MySQL, Redis
- 架构：微服务, 高并发, 分布式
- 工具：Docker, Kubernetes, 消息队列

**步骤2：将关键词自然融入简历**

```markdown
## 技能
- 编程语言：Java, Python, Go
- 框架：**Spring Boot**, Spring Cloud, MyBatis
- 数据库：**MySQL**, PostgreSQL, MongoDB
- 缓存：**Redis**, Memcached
- 消息队列：Kafka, RabbitMQ, **消息队列**
- 容器化：**Docker**, **Kubernetes**
- 架构：**微服务**, **高并发**, 分布式系统
```

**关键词密度原则**：
- 主要关键词出现2-3次
- 分布在不同模块（技能、经历、项目）
- 不要堆砌，保持自然

---

### 4. 技能清单写法

#### ❌ 错误示例
```
技能：
- 熟悉各种编程语言
- 掌握常用框架
- 了解数据库
```

#### ✅ 正确示例
```markdown
## 专业技能

**编程语言**
Java (熟练), Python (熟练), Go (掌握), JavaScript (掌握)

**后端框架**
Spring Boot, Spring Cloud, MyBatis, Netty, gRPC

**数据库**
MySQL (索引优化、分库分表), MongoDB, PostgreSQL, Redis

**中间件**
Kafka, RabbitMQ, RocketMQ, Zookeeper

**DevOps**
Docker, Kubernetes, Jenkins, Git, CI/CD

**其他**
微服务架构, 分布式系统, 高并发设计, 系统性能优化
```

**技巧**：
- 按类别分组，便于ATS抓取
- 标注熟练程度（可选）
- 使用标准术语，不用自造词

## 🎯 不同公司的侧重点

### 大厂（BAT、TMD等）
**关注点**：技术深度、系统设计、解决问题能力

**简历调整**：
- 强调技术栈的深度（如：深入理解JVM、有性能调优经验）
- 突出系统设计能力（架构图、技术方案）
- 量化结果（QPS、延迟、可用性等指标）
- 列出有挑战性的技术难题及解决方案

### 创业公司
**关注点**：全栈能力、快速交付、主人翁精神

**简历调整**：
- 强调技术广度（全栈、多语言）
- 突出产品意识和业务理解
- 展示快速学习能力（新技能、新项目）
- 提及创业/副项目经历

### 外企
**关注点**：沟通能力、英语、团队协作

**简历调整**：
- 准备英文简历版
- 强调跨部门协作经验
- 突出国际化背景（如有）
- 展示软技能（领导力、 mentoring）

## ⚠️ 常见错误

| 错误类型 | 问题描述 | 后果 |
|---------|---------|------|
| 格式混乱 | 使用复杂排版、表格、图表 | ATS无法正确解析 |
| 关键词缺失 | 不匹配JD关键词 | 被自动过滤 |
| 模糊表达 | "负责...""参与..." | 无法突出贡献 |
| 格式不一致 | 日期格式不统一 | 显得不专业 |
| 错别字 | 简历中有拼写错误 | 显得粗心 |

## 📊 简历检查清单

提交前对照检查：

```
□ 文件格式正确（PDF或Word）
□ 联系方式清晰可见
□ 包含JD中的关键词
□ 工作经历倒序排列
□ 每段经历有量化结果
□ 日期格式一致（如：2020.06 - 2022.02）
□ 没有拼写/语法错误
□ 文件命名清晰（如：张三_后端工程师_5年经验.pdf）
□ 在不同设备上检查格式是否正常
□ 请朋友帮忙审阅
```
